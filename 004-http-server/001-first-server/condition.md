# Модуль 1. Первый HTTP-сервер — условие

## Задание

Написать простой HTTP-сервер с **тремя эндпоинтами**, который ты сможешь потрогать через `curl` и через браузер. Никаких задач, никакого JSON — только знакомство с механикой `net/http`.

## Что должна делать программа

```bash
$ go run .
Сервер запущен на http://localhost:8080
```

В другом терминале:

```bash
$ curl http://localhost:8080/hello
Hello, world!

$ curl http://localhost:8080/ping
pong

$ curl http://localhost:8080/info
Метод: GET
Путь: /info
User-Agent: curl/7.68.0

$ curl http://localhost:8080/anything-else
404 page not found
```

## Требования

1. Создай **новую папку** `project-4-http-server/practice-server/` для этого модуля
2. Эндпоинт `/hello` — отвечает строкой `"Hello, world!"`
3. Эндпоинт `/ping` — отвечает строкой `"pong"`
4. Эндпоинт `/info` — отвечает тремя строками: метод запроса, путь, заголовок `User-Agent`
5. На любой другой путь — Go автоматически вернёт 404 (твоя задача — **ничего** для этого специально не делать)
6. Сервер слушает порт **8080**
7. При запуске выводит сообщение «Сервер запущен на http://localhost:8080»
8. Если `ListenAndServe` вернёт ошибку — выводится сообщение
9. Каждый обработчик — **отдельная функция** (не лямбда)

## Шаги

1. Создай папку, инициализируй модуль (`go mod init server`)
2. Создай `main.go`
3. Импортируй `fmt` и `net/http`
4. Напиши функцию `helloHandler(w http.ResponseWriter, r *http.Request)`
5. Внутри — `fmt.Fprintln(w, "Hello, world!")`
6. Аналогично — `pingHandler` и `infoHandler`
7. В `main`:
   - Вызови `http.HandleFunc("/hello", helloHandler)` для каждого эндпоинта
   - Выведи сообщение о запуске
   - Вызови `http.ListenAndServe(":8080", nil)`
   - Обработай ошибку
8. Запусти `go run .`
9. В другом терминале проверь все три эндпоинта через `curl`
10. Открой `http://localhost:8080/hello` в браузере — тоже должно работать

## Эксперименты

1. **Открой `curl -v http://localhost:8080/hello`.** Увидишь HTTP-запрос и ответ полностью — заголовки, статус, тело. Прочитай вывод, попробуй понять, что есть что. Это та самая «магия», которая на самом деле просто текст.

2. **Останови сервер (Ctrl+C) и запусти заново.** Скорость запуска — практически мгновенная. Это хорошо: цикл «изменил код → перезапустил → проверил» очень быстрый.

3. **Запусти сервер и попробуй запустить ВТОРОЙ экземпляр** в другом терминале. Что произошло? Прочитай ошибку. Почему так? (Подсказка: два процесса не могут слушать один и тот же порт.)

4. **Поменяй порт на 8081**, перезапусти, проверь, что работает.

5. **Поменяй порт на 80** (без `sudo`). Что произошло? Прочитай ошибку. Это про права доступа: порты ниже 1024 — привилегированные.

6. **Попробуй POST-запрос на свой эндпоинт:**
   ```bash
   curl -X POST http://localhost:8080/hello
   ```
   Что произошло? Сервер ответил тем же `Hello, world!`. Это потому, что наш handler **не проверяет метод** — он отвечает на любой запрос к `/hello`. В модулях 2-4 мы научимся различать методы.

7. **Запусти сервер и подключись через telnet:**
   ```bash
   telnet localhost 8080
   ```
   После подключения набери:
   ```
   GET /hello HTTP/1.1
   Host: localhost
   
   ```
   (Двойной Enter в конце!) Увидишь сырой HTTP-ответ. Это то, что обычно делает за тебя браузер. **Это самый важный эксперимент модуля** — он покажет тебе, что HTTP — это просто текст.

   На macOS `telnet` может быть не установлен, тогда используй `nc`:
   ```bash
   nc localhost 8080
   ```
   и набери то же самое.

## Что показать Claude

Покажи Claude свой код и спроси: «что произойдёт, если в `helloHandler` вызвать `WriteHeader(500)` после `Fprintln(w, "Hello")`?». Это ловушка: статус **должен** быть установлен **до** записи в тело. Если ты сделаешь наоборот, Go выведет предупреждение в логи (`http: superfluous response.WriteHeader call`) и проигнорирует второй вызов. Понимание этой последовательности важно для следующих модулей.
