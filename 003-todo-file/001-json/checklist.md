# Модуль 1. JSON — чек-лист готовности

## Поведенческие маркеры

- [ ] Создана отдельная папка `practice-json/` с собственным `go.mod`
- [ ] Объявлен тип `Task` с тремя полями (ID, Title, Done)
- [ ] У каждого поля есть JSON-тег с именем в нижнем регистре
- [ ] Программа выводит JSON одной задачи через `MarshalIndent` с двумя пробелами отступа
- [ ] Программа десериализует обратно и выводит результат через `Printf("%+v\n", ...)`
- [ ] То же самое делается для слайса из трёх задач
- [ ] Все ошибки от `Marshal` и `Unmarshal` проверяются явно
- [ ] Программа компилируется и запускается без ошибок

## Концептуальные маркеры

- [ ] Могу объяснить, что такое **сериализация** и зачем она нужна
- [ ] Знаю, что JSON — это **текстовый** формат (могу прочитать глазами)
- [ ] Перечисляю четыре простых типа JSON и две структуры
- [ ] Знаю, какой пакет в Go отвечает за JSON
- [ ] Понимаю разницу между `Marshal` и `MarshalIndent` — и когда какой использовать
- [ ] Знаю, что `Marshal` возвращает `[]byte`, и зачем конвертировать через `string(...)`
- [ ] Понимаю, **почему** `Unmarshal` принимает указатель
- [ ] Могу объяснить, **зачем** нужны JSON-теги, если без них тоже работает
- [ ] Знаю, что **приватные** поля (с маленькой буквы) **не сериализуются**
- [ ] Знаю про опции `omitempty` и `-` в тегах

## Маркеры экспериментов

- [ ] Убрал теги, увидел, что имена в JSON стали с большой буквы
- [ ] Сделал поле приватным, увидел, что оно пропало из JSON
- [ ] Попробовал `omitempty` — увидел разницу для пустого и непустого значения
- [ ] Сравнил `Marshal` и `MarshalIndent` визуально
- [ ] Подсунул битый JSON в `Unmarshal`, прочитал сообщение об ошибке
- [ ] Подсунул JSON с неправильным типом, прочитал сообщение
- [ ] Подсунул JSON с лишним полем, заметил, что оно тихо игнорируется

## Самопроверка

Не подсматривая, ответь:

1. Как выглядит синтаксис JSON-тега? (запиши на бумаге одну строку структуры с тегом)
2. Какие два аргумента у `MarshalIndent`, помимо самого значения?
3. Что нужно передать `Unmarshal` — значение или адрес? Почему?
4. Что вернёт `Marshal`, если у структуры все поля приватные?

Если на все четыре отвечаешь уверенно — модуль закрыт.

## Подготовка к модулю 2

В модуле 2 ты возьмёшь JSON-байты, которые получил здесь, и научишься **записывать их в файл**. А потом — **читать** обратно. Это ровно те же байты, только теперь они путешествуют между программой и диском.

Главный вопрос для размышления перед модулем 2: **что делать, если файла нет?** При первом запуске программы файла `tasks.json` ещё не существует. Это **ошибка** или **нормальная ситуация**? Подумай — это не риторический вопрос, это будет частью задания.
