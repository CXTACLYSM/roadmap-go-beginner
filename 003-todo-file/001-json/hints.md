## Уровень 1: наводящие вопросы

1. Какой пакет в стандартной библиотеке Go отвечает за JSON?
2. Какая функция превращает Go-структуру в JSON-байты?
3. Какая делает обратное?
4. Какого типа результат у `Marshal` — `string` или `[]byte`?
5. Что нужно передать второму аргументу `Unmarshal` — значение или указатель? Почему?

## Уровень 2: про синтаксис тегов

JSON-теги пишутся **в обратных кавычках** сразу после типа поля:

```go
type Task struct {
    ID    int    `json:"id"`
    Title string `json:"title"`
    Done  bool   `json:"done"`
}
```

Несколько важных моментов:

- Кавычки — **обратные** (символ ` `, обычно слева от `1` на клавиатуре), не одинарные и не двойные
- Внутри обратных кавычек — сначала имя ключа (`json`), потом двоеточие, потом значение в обычных двойных кавычках
- Между двоеточием и значением **нет пробела**: `json:"id"`, не `json: "id"`
- Без пробелов и переносов строк внутри тега

## Уровень 3: про вывод []byte как текст

`Marshal` возвращает `[]byte` (массив байт). Если ты попытаешься напечатать его напрямую:

```go
data, _ := json.Marshal(task)
fmt.Println(data)
// [123 34 105 100 ...]
```

— увидишь массив чисел. Это потому, что `Println` для `[]byte` печатает байты как числа, а не как символы.

Чтобы получить читаемую строку — конвертируй:

```go
fmt.Println(string(data))
```

`string(data)` — это **конвертация типов**: «возьми эти байты и интерпретируй их как строку». Работает мгновенно и без аллокаций (внутри строки в Go и так лежат байты).

## Уровень 4: про MarshalIndent

```go
data, err := json.MarshalIndent(task, "", "  ")
//                                    ^^  ^^^^
//                                    |   |
//                              prefix    indent
```

- **`prefix`** — добавляется в начало **каждой строки**. Обычно пустая строка `""`.
- **`indent`** — что использовать для отступа. Стандарт — два пробела `"  "` или таб `"\t"`.

Если оставить оба пустыми — `MarshalIndent` ведёт себя как `Marshal` (одна строка без отступов).

## Уровень 5: про указатель в Unmarshal

```go
var task Task                      // объявляем переменную
err := json.Unmarshal(data, &task) // передаём её АДРЕС
```

Та же логика, что у `fmt.Scan` из Проекта 1. `Unmarshal` должен **изменить** переменную — записать в неё распарсенные данные. Чтобы изменить — нужен указатель на саму переменную, а не её копию.

Если забудешь `&`:

```
cannot use task (variable of type Task) as type *Task in argument to json.Unmarshal
```

Расшифровка: «нужен указатель на Task, а не сам Task».

## Уровень 6: про слайс в Unmarshal

Для слайса то же самое — нужен указатель на переменную-слайс:

```go
var tasks []Task
err := json.Unmarshal(data, &tasks)
```

`tasks` сначала `nil`, после `Unmarshal` будет содержать распарсенные элементы. Никакого `make` или `append` не нужно — `Unmarshal` всё сделает сам.

## Уровень 7: про %+v в Printf

Для отладки структур есть полезный глагол `%+v`:

```go
fmt.Printf("%+v\n", task)
// {ID:1 Title:Купить молоко Done:false}
```

В отличие от `%v` (просто значения через пробел), `%+v` показывает **имена полей**. Очень полезно, когда у тебя структура с несколькими полями и ты не помнишь порядок.

## Уровень 8: про обработку ошибок

`Marshal` и `Unmarshal` возвращают `error`. Игнорировать его — плохая привычка с самого начала. Вот шаблон:

```go
data, err := json.Marshal(task)
if err != nil {
    fmt.Println("Ошибка сериализации:", err)
    return
}
```

`Marshal` обычно проваливается, только если в структуре есть какие-то совсем странные поля (циклические указатели, например). Для обычных задач он почти всегда успешен. Но привычка проверять — обязательна.

`Unmarshal` проваливается часто: битый JSON, несоответствие типов, неполные данные. Здесь проверка `err` критична.

Скажи Claude: «модуль 1 проекта 3, JSON, проблема: [конкретное описание]». Покажи код, опиши, что ожидал, что получил. Самые частые ситуации:

- «Поле не попадает в JSON» → ищем поле с маленькой буквы или забытый тег
- «Unmarshal не работает» → ищем отсутствие `&` перед переменной
- «Странный вывод массива чисел» → забыл `string(data)` при печати
