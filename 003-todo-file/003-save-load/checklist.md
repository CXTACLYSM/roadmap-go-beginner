## Самопроверка: финальный сценарий

Прогони этот сценарий и убедись, что **всё** работает:

```bash
# 1. Удаляем файл
rm -f tasks.json

# 2. Первый запуск
go run .
> add Купить молоко
> add Позвонить маме
> done 1
> exit

# 3. Проверяем файл
cat tasks.json
# должен быть отформатированный JSON с двумя задачами,
# первая с done: true

# 4. Второй запуск
go run .
> list
# должны быть обе задачи, первая с [x]
> add Сделать домашку
# должна быть #3, не #1
> remove 2
> exit

# 5. Третий запуск
go run .
> list
# должны быть две задачи: #1 (выполненная) и #3
> exit
```

Если все три запуска прошли без проблем — модуль закрыт.

## Подготовка к модулю 4

К этому моменту твой `main.go` стал довольно большим: тип `Task`, тип `TaskList` со всеми методами, `Save`/`Load`, handle-функции, REPL, helper-ы. Прикинь длину — наверняка 200+ строк.

В модуле 4 ты разложишь этот код на **несколько файлов** — это первое знакомство с организацией кода в Go-проектах. Никаких новых концепций, только перекладывание. Это будет приятная разминка после интенсивных модулей 2 и 3.

Подумай заранее: **какие группы кода** ты бы выделил в отдельные файлы? Какие функции логически связаны и должны жить рядом? Это упражнение разовьёт твою интуицию про **модульность**.
