## Уровень 2: про объявление слайса

Самые частые формы:

```go
var tasks []Task                // пустой, len=0, cap=0
tasks := []Task{}               // то же самое, через :=
tasks := []Task{                // сразу с элементами
    {ID: 1, Title: "first"},
    {ID: 2, Title: "second"},
}
```

Все три валидные. Выбирай по контексту:
- `var tasks []Task` — когда будешь заполнять через `append`
- литерал с элементами — когда знаешь данные сразу

## Уровень 3: про append с присваиванием

Если ты пишешь так:

```go
append(tasks, newTask)
```

— код **скомпилируется** (потому что `append` возвращает значение, а в Go можно игнорировать возвращаемые значения от выражений), но **новый элемент потеряется**. Ты создал новый слайс и тут же его выбросил.

Правильно:

```go
tasks = append(tasks, newTask)
```

Старый `tasks` заменяется на новый, в котором есть новый элемент.

**Запомни эту ошибку.** Это один из самых частых багов у новичков, и компилятор не предупредит — программа просто работает не так, как ты ожидаешь.

## Уровень 4: про range с двумя переменными

Синтаксис:

```go
for index, value := range slice {
    // ...
}
```

`range` для слайса всегда возвращает **два** значения: индекс и копию элемента. Ты можешь:

- Использовать оба: `for i, t := range tasks`
- Игнорировать индекс: `for _, t := range tasks`
- Игнорировать значение: `for i := range tasks` (один параметр — это автоматически индекс)

## Уровень 5: про **главную ловушку** range

Если ты пишешь:

```go
for _, t := range tasks {
    t.Done = true
}
```

— **изменения не сохранятся**. Это потому, что `t` — это **копия** элемента, а не сам элемент.

Аналогия из модуля 1: помнишь, как `MarkDone` с receiver-значением не работал, потому что `t` было копией? Здесь то же самое: `range` копирует каждый элемент в локальную переменную итерации.

**Правильно — изменять напрямую через индекс:**

```go
for i := range tasks {
    tasks[i].Done = true
}
```

Здесь `tasks[i]` — это не копия, а **прямой доступ** к элементу слайса. Изменение видно снаружи цикла.

> Если эта концепция кажется странной — ты на правильном пути. Это одна из самых концептуально важных вещей в Go. Проведи эксперимент 1 из condition.md, почувствуй разницу руками.

## Уровень 6: про индексы и границы

Слайс из 3 элементов имеет индексы `0`, `1`, `2`. Обращение к `tasks[3]` или `tasks[10]` — это **выход за границы**, и Go в таком случае **паникует**:

```
panic: runtime error: index out of range [10] with length 3
```

Это значит: программа была остановлена в аварийном режиме. Поэтому всегда — **либо проверяй длину**, либо **используй range** (который автоматически не выйдет за границы).

## Уровень 7: как пометить первую задачу

Тебе нужно что-то типа:

```go
tasks[0].Done = true
```

— это самый простой вариант (если знаешь, что первая задача — индекс 0). Или через поиск по ID:

```go
for i := range tasks {
    if tasks[i].ID == 1 {
        tasks[i].Done = true
        break
    }
}
```

Заметь: оба варианта обращаются через индекс, а не через копию из range. Это и есть правильный путь.
