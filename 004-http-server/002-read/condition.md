# Модуль 2. GET — чтение задач — условие

## Задание

Создать сервер с **двумя GET-эндпоинтами**: списка задач и одной задачи по ID. Использовать тип `Server` со ссылкой на `TaskList`. Данные пока **захардкожены в коде** — создание через POST будет в модуле 3.

## Что должна делать программа

```bash
$ go run .
Сервер запущен на http://localhost:8080

# В другом терминале:
$ curl -i http://localhost:8080/tasks
HTTP/1.1 200 OK
Content-Type: application/json
...

[{"id":1,"title":"Купить молоко","done":false},{"id":2,"title":"Позвонить маме","done":true}]

$ curl -i http://localhost:8080/tasks/1
HTTP/1.1 200 OK
Content-Type: application/json
...

{"id":1,"title":"Купить молоко","done":false}

$ curl -i http://localhost:8080/tasks/999
HTTP/1.1 404 Not Found
Content-Type: text/plain; charset=utf-8
...

задача не найдена: 999

$ curl -i http://localhost:8080/tasks/abc
HTTP/1.1 400 Bad Request
...

ID должен быть числом

$ curl -i -X POST http://localhost:8080/tasks
HTTP/1.1 405 Method Not Allowed
...
```

## Требования

1. Создай папку `app/` — это основной код Проекта 4. Перенеси туда `Task` и `TaskList` из Проекта 3 (можешь упростить — убери Save/Load пока, добавим в модуле 6).
2. Тип `TaskList` имеет публичные поля `Tasks []Task` и `NextID int` с JSON-тегами.
3. Добавь метод `GetByID(id int) (Task, error)` — возвращает задачу или ошибку «не найдена».
4. Создай тип `Server`:
   ```go
   type Server struct {
       tasks *TaskList
   }
   ```
5. Реализуй два метода-обработчика:
   - `(s *Server) listTasks(w, r)` — возвращает все задачи как JSON
   - `(s *Server) getTask(w, r)` — возвращает одну задачу по ID
6. В `main`:
   - Создаётся `Server` с **двумя захардкоженными задачами**
   - Регистрируются обработчики через `http.HandleFunc("GET /tasks", ...)` и `("GET /tasks/{id}", ...)`
   - Выводится сообщение о запуске
   - Запускается `http.ListenAndServe(":8080", nil)`
7. **Метод указывается прямо в pattern-е** — `"GET /tasks"`, не отдельная проверка `r.Method`.
8. Для парсинга ID — `strconv.Atoi(r.PathValue("id"))`.
9. Заголовок `Content-Type: application/json` устанавливается перед записью JSON.
10. Используется `json.NewEncoder(w).Encode(...)`, не `Marshal + Write`.
11. Все ошибки обрабатываются с правильными HTTP-статусами (400, 404, 500).
12. После каждой `http.Error(...)` стоит `return`.
13. Запросы с **другими методами** на эти пути возвращают 405 (это Go делает автоматически для path patterns с явным методом).

## Эксперименты

1. **Запусти `curl -i`** для каждого случая. Прочитай заголовки ответа. Особенно — `Content-Type` для успешных и для ошибок (они **разные**).

2. **Открой `http://localhost:8080/tasks` в браузере.** Современные браузеры умеют красиво показывать JSON прямо во вкладке. Если у тебя Chrome/Firefox — есть встроенная подсветка.

3. **Забудь установить Content-Type** в успешном ответе. Запусти, проверь через браузер. JSON всё ещё показывается, но как обычный текст. Сервер работает, но «теряет дружелюбность» к клиенту — теперь клиенту приходится угадывать формат.

4. **Забудь `return` после `http.Error`.** Запусти, попроси несуществующую задачу. В логе сервера увидишь `http: superfluous response.WriteHeader call`. Понимаешь, почему?

5. **Используй `json.Marshal` + `w.Write` вместо `Encoder.Encode`.** Сравни — на маленьких данных разницы не видно. Какой вариант тебе удобнее? Какой более идиоматичен в Go?

6. **Запросы с разными методами:**
   ```bash
   curl -X POST http://localhost:8080/tasks
   curl -X PUT http://localhost:8080/tasks
   curl -X DELETE http://localhost:8080/tasks
   ```
   Все должны вернуть **405 Method Not Allowed**. Это потому, что у тебя зарегистрирован только `GET /tasks`. Раньше (до Go 1.22) пришлось бы проверять метод вручную в обработчике.

7. **Зарегистрируй ещё один обработчик с тем же путём, но другим методом:**
   ```go
   http.HandleFunc("POST /tasks", func(w http.ResponseWriter, r *http.Request) {
       fmt.Fprintln(w, "POST received")
   })
   ```
   Теперь POST работает, GET — тоже. Это путь к полноценному CRUD, который ты сделаешь в модулях 3-4.

8. **Попробуй несколько path-параметров:**
   ```go
   http.HandleFunc("GET /users/{userID}/tasks/{taskID}", ...)
   ```
   Извлекаешь оба через `r.PathValue("userID")` и `r.PathValue("taskID")`. Это понадобится в более сложных API.

## Что показать Claude

Покажи Claude свой `getTask` и спроси: «правильно ли я различаю ошибки 400 и 404? Какая логика у каждой?». Это **дизайнерский** вопрос, на который у разных команд могут быть разные ответы. Например, некоторые компании возвращают **404** и для нечислового ID (мол, «такого ресурса нет — и не может быть»), а другие — **400** (мол, «ты неправильно сформулировал запрос»). Оба подхода валидны, но **в одном проекте надо быть последовательным**.
