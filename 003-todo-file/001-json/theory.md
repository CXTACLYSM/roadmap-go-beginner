# Модуль 1. JSON — теория

## Контекст

В Проектах 1 и 2 твоя программа работала с данными **только в памяти**. Структуры жили в переменных, перемещались между функциями, менялись и удалялись. Всё это происходило **внутри** одной запущенной программы, и за её пределы не выходило.

Но что если данные нужно:

- **Сохранить на диск**, чтобы вернуться к ним завтра?
- **Передать по сети** на другой компьютер?
- **Показать другой программе**, написанной на другом языке?

Во всех этих случаях возникает одна и та же задача: превратить **структуру в памяти** в **последовательность байтов**, которую можно записать или передать. И обратно — из последовательности байтов восстановить структуру.

Этот процесс называется **сериализация** (превращение в последовательность) и **десериализация** (обратно). А JSON — самый популярный **формат**, в котором это делают.

## Ядро: что такое JSON

JSON (**J**ava**S**cript **O**bject **N**otation) — это текстовый формат для описания структурированных данных. Он выглядит так:

```json
{
  "id": 1,
  "title": "Купить молоко",
  "done": false
}
```

Это та же `Task`, что у тебя в Go, только в виде текста. JSON поддерживает четыре простых типа и две структуры:

| Тип | Пример |
|-----|--------|
| Число | `42`, `3.14` |
| Строка | `"привет"` |
| Булевое | `true`, `false` |
| `null` | `null` |
| Массив | `[1, 2, 3]` |
| Объект | `{"key": "value"}` |

И всё. Этого достаточно для описания почти любых данных.

## Почему JSON

JSON выиграл рынок по трём причинам:

1. **Читается человеком.** В отличие от бинарных форматов, JSON можно открыть в любом текстовом редакторе и понять, что внутри.
2. **Просто парсится машиной.** Любой язык программирования из коробки умеет работать с JSON. Структура простая, парсеры быстрые.
3. **Поддерживается везде.** Веб-API, конфиги, базы данных, очереди сообщений — JSON везде. Если ты выучил JSON, ты выучил «общий язык» современного бэкенда.

В Go для работы с ним есть стандартный пакет `encoding/json`. Никаких внешних зависимостей.

## Структура: Marshal и Unmarshal

Две главные функции, которые нужно знать:

```go
import "encoding/json"

// Структура → JSON-байты (сериализация)
data, err := json.Marshal(task)

// JSON-байты → структура (десериализация)
var task Task
err := json.Unmarshal(data, &task)
```

**`Marshal`** — берёт значение и возвращает `[]byte` с JSON-представлением.

**`Unmarshal`** — берёт `[]byte` и **указатель** на переменную, в которую нужно записать результат. Это та же логика, что у `fmt.Scan`: функция должна **изменить** переменную, поэтому нужен указатель.

Обе функции возвращают `error` — на случай, если что-то не так с данными или JSON.

## Пример: одна задача

```go
package main

import (
    "encoding/json"
    "fmt"
)

type Task struct {
    ID    int
    Title string
    Done  bool
}

func main() {
    task := Task{ID: 1, Title: "Купить молоко", Done: false}

    data, err := json.Marshal(task)
    if err != nil {
        fmt.Println("Ошибка сериализации:", err)
        return
    }

    fmt.Println(string(data))
}
```

Вывод:

```json
{"ID":1,"Title":"Купить молоко","Done":false}
```

Несколько важных моментов:

1. **`Marshal` вернул `[]byte`**, а не `string`. JSON — это именно последовательность байт. Чтобы напечатать как текст, мы конвертируем через `string(data)`.
2. **Имена полей в JSON совпадают с именами в Go** — `ID`, `Title`, `Done`. С большой буквы. Это **некрасиво** для JSON: в индустрии принят `lowercase` или `camelCase`.
3. **Только публичные поля** (с большой буквы) попадают в JSON. Если поле приватное (с маленькой буквы), `json.Marshal` его проигнорирует. Это касается и `Unmarshal` — приватные поля не заполнятся.

## JSON-теги: контроль над сериализацией

Чтобы поправить имена полей в JSON, не меняя имён в Go, используют **теги** — специальные метаданные, которые пишутся **в обратных кавычках** после типа поля:

```go
type Task struct {
    ID    int    `json:"id"`
    Title string `json:"title"`
    Done  bool   `json:"done"`
}
```

Теперь `Marshal` выдаст:

```json
{"id":1,"title":"Купить молоко","done":false}
```

Имена полей в Go остались как были (`ID`, `Title`, `Done` — с большой буквы, чтобы быть публичными). Имена в JSON — какие захотел через тег.

Это **очень частая идиома**. Почти все Go-структуры, которые сериализуются в JSON, имеют такие теги.

### Дополнительные опции тегов

```go
type Task struct {
    ID    int    `json:"id"`
    Title string `json:"title"`
    Done  bool   `json:"done,omitempty"`     // не включать, если false/нулевое
    Notes string `json:"notes,omitempty"`    // не включать, если пустая строка
    secret string `json:"-"`                  // не включать никогда
}
```

`omitempty` — «опусти, если пустое». Полезно для опциональных полей.
`-` (дефис) — «никогда не сериализуй это поле». Полезно для внутренних данных.

## MarshalIndent: красивый вывод

`Marshal` выдаёт всё в одну строку — компактно, но не очень читаемо для человека. Для красивого вывода с отступами есть `MarshalIndent`:

```go
data, err := json.MarshalIndent(task, "", "  ")
```

Третий параметр — **строка отступа**. Два пробела — стандарт. Результат:

```json
{
  "id": 1,
  "title": "Купить молоко",
  "done": false
}
```

Когда использовать `MarshalIndent`, а когда `Marshal`:

- **`Marshal`** — для передачи по сети, для записи в файл, который читает другая программа. Минимум места.
- **`MarshalIndent`** — для записи в файл, который будут читать **люди** (конфиги, отладочные дампы, наш `tasks.json` в этом проекте).

## Десериализация: Unmarshal

Обратный процесс:

```go
data := []byte(`{"id":2,"title":"Позвонить маме","done":true}`)

var task Task
err := json.Unmarshal(data, &task)
if err != nil {
    fmt.Println("Ошибка парсинга:", err)
    return
}

fmt.Printf("%+v\n", task)
// {ID:2 Title:Позвонить маме Done:true}
```

Заметь:

1. `data` — это `[]byte`. Если у тебя строка — конвертируй через `[]byte(s)`.
2. `&task` — указатель на переменную. `Unmarshal` запишет туда результат.
3. `%+v` в `Printf` — формат «вывести структуру с именами полей». Полезно для отладки.

### Что произойдёт, если JSON не подходит

Несколько типичных случаев:

**Лишние поля в JSON** — игнорируются. Если у тебя в Go нет поля `priority`, а в JSON оно есть, `Unmarshal` его просто не заметит. Это удобно для совместимости версий.

**Поля Go отсутствуют в JSON** — остаются с **нулевыми значениями**. Если в JSON нет `done`, то `task.Done` будет `false` после `Unmarshal`. Не ошибка, просто значение по умолчанию.

**Несовпадение типов** — `error`. Если в JSON `"id": "abc"` (строка), а в Go `ID int` — `Unmarshal` вернёт ошибку, потому что строку нельзя положить в `int`.

**Битый JSON** (синтаксическая ошибка) — `error`. Например, потерянная скобка или запятая.

**Всегда проверяй `err`** после `Unmarshal`. Это твой главный защитник от мусорных данных.

## Сериализация слайсов и сложных структур

`Marshal` умеет работать не только со структурами, но и со слайсами, мапами, и всем, что из них состоит:

```go
tasks := []Task{
    {ID: 1, Title: "первая", Done: false},
    {ID: 2, Title: "вторая", Done: true},
}

data, _ := json.MarshalIndent(tasks, "", "  ")
fmt.Println(string(data))
```

Вывод:

```json
[
  {
    "id": 1,
    "title": "первая",
    "done": false
  },
  {
    "id": 2,
    "title": "вторая",
    "done": true
  }
]
```

Слайс становится JSON-массивом. Каждый элемент — отдельный объект. Просто и предсказуемо.

И обратно:

```go
var loaded []Task
err := json.Unmarshal(data, &loaded)
```

Здесь `loaded` — это слайс (нулевое значение — `nil`), и `Unmarshal` его наполнит.

## Полный пример

```go
package main

import (
    "encoding/json"
    "fmt"
)

type Task struct {
    ID    int    `json:"id"`
    Title string `json:"title"`
    Done  bool   `json:"done"`
}

func main() {
    // Создаём слайс задач
    tasks := []Task{
        {ID: 1, Title: "Купить молоко", Done: false},
        {ID: 2, Title: "Позвонить маме", Done: true},
    }

    // Сериализуем в красивый JSON
    data, err := json.MarshalIndent(tasks, "", "  ")
    if err != nil {
        fmt.Println("Ошибка сериализации:", err)
        return
    }

    fmt.Println("--- JSON ---")
    fmt.Println(string(data))

    // Десериализуем обратно
    var loaded []Task
    err = json.Unmarshal(data, &loaded)
    if err != nil {
        fmt.Println("Ошибка десериализации:", err)
        return
    }

    fmt.Println("--- Восстановлено ---")
    for _, t := range loaded {
        fmt.Printf("%+v\n", t)
    }
}
```
