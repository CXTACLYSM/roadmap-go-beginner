# Модуль 1. Структура проекта — чек-лист готовности

## Поведенческие маркеры

### Структура

- [ ] Создан корневой `go.mod` с module path
- [ ] Создана папка `cmd/server/` с `main.go`
- [ ] Создана папка `internal/api/` с минимум одним файлом
- [ ] Создана папка `internal/auth/` с минимум одним файлом
- [ ] Создана папка `internal/config/` с минимум одним файлом
- [ ] Создана папка `internal/domain/` с минимум одним файлом
- [ ] Создана папка `internal/storage/` с минимум одним файлом
- [ ] Создана пустая папка `migrations/`
- [ ] Создан файл `.env.example` с тремя переменными
- [ ] Имена пакетов совпадают с именами папок

### Код

- [ ] В `internal/config/`: тип `Config` с полями `DatabaseURL`, `JWTSecret`, `Port`
- [ ] В `internal/config/`: функция `Load() (*Config, error)`
- [ ] `Load` возвращает ошибку при отсутствии `DATABASE_URL`
- [ ] `Load` возвращает ошибку при отсутствии `JWT_SECRET`
- [ ] `Load` устанавливает `Port = "8080"` при пустом значении
- [ ] В `internal/domain/`: тип `User` с полями `ID`, `Email`, `CreatedAt` и JSON-тегами
- [ ] В `internal/storage/`: тип `UserRepository` с полем `db *sql.DB`
- [ ] В `internal/storage/`: конструктор `NewUserRepository`
- [ ] В `internal/auth/`: тип `Service` с полем `secret string`
- [ ] В `internal/auth/`: конструктор `NewService`
- [ ] В `internal/api/`: тип `Handler` с приватными полями `users` и `auth`
- [ ] В `internal/api/`: конструктор `NewHandler`
- [ ] В `internal/api/`: метод `RegisterRoutes(mux *http.ServeMux)`
- [ ] В `internal/api/`: метод `hello(w, r)`
- [ ] В `cmd/server/main.go`: вызов `config.Load()` с обработкой ошибки
- [ ] В `main.go`: создание зависимостей через конструкторы
- [ ] В `main.go`: создан **свой** `mux := http.NewServeMux()`
- [ ] В `main.go`: `handler.RegisterRoutes(mux)`
- [ ] В `main.go`: `http.ListenAndServe(":"+cfg.Port, mux)`

### Запуск

- [ ] `go run ./cmd/server` запускается с правильными env
- [ ] `curl http://localhost:8080/hello` возвращает «Hello, blog!»
- [ ] При отсутствии `DATABASE_URL` — выход с ошибкой
- [ ] При отсутствии `JWT_SECRET` — выход с ошибкой
- [ ] `go vet ./...` проходит без ошибок
- [ ] `go fmt ./...` ничего не меняет (или ты применил его)
- [ ] Код закоммичен
