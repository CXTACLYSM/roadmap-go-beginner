# Проект 2. To-Do лист в памяти

**Длительность:** 3 недели
**Цель:** освоить структуры, слайсы и методы — три кита, на которых стоит весь Go. Научиться писать программы с состоянием, которое меняется во времени.

## Что в итоге будет уметь программа

Консольное приложение, которое после запуска ждёт команды:

```
> add Купить молоко
Задача добавлена: #1 Купить молоко

> add Позвонить маме
Задача добавлена: #2 Позвонить маме

> list
1. [ ] Купить молоко
2. [ ] Позвонить маме

> done 1
Задача #1 отмечена как выполненная

> list
1. [x] Купить молоко
2. [ ] Позвонить маме

> remove 2
Задача #2 удалена

> exit
До встречи!
```

Данные живут только пока программа запущена. Перезапустил — список пустой. Сохранение в файл будет в Проекте 3.

## Что нового по сравнению с Проектом 1

- **Структуры** — собственные типы данных. Это твой первый шаг к настоящему программированию «в объектах» (хотя в Go нет классов).
- **Методы** — функции, привязанные к типу. Идиоматичный Go без них непредставим.
- **Слайсы изнутри** — длина vs ёмкость, поведение `append` при росте, ловушки с разделяемой памятью. Самая концептуально сложная тема первых проектов.
- **REPL** — Read-Eval-Print Loop. Шаблон долгоживущей интерактивной программы. Понадобится потом для серверов.
- **Состояние программы** — впервые у тебя будут данные, которые **меняются** во времени. Это требует другого мышления, чем «принял аргументы — посчитал — вышел».

## Главная ловушка этого проекта

В Проекте 1 ты привык, что программа — это «функция, которая что-то считает и завершается». В этом проекте программа **живёт** во времени. У неё есть **состояние** (список задач), которое меняется от команды к команде.

Это требует другого мышления. Многие новички пытаются по-старому: «вот я добавил задачу — а теперь она пропала». Запомни: **состояние нужно где-то хранить между итерациями цикла**, и это «где-то» — переменная, объявленная **до** цикла.
