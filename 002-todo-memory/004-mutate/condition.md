# Модуль 4. Изменение и удаление — условие

## Задание

Добавить в REPL **две новые команды**: `done <ID>` и `remove <ID>`. Первая отмечает задачу как выполненную, вторая удаляет её из списка. Обе принимают ID задачи, должны корректно реагировать на ошибочный ввод.

## Что должна делать программа

```
> add Купить молоко
Задача добавлена: #1 Купить молоко
> add Позвонить маме
Задача добавлена: #2 Позвонить маме
> add Сделать домашку
Задача добавлена: #3 Сделать домашку
> list
1. [ ] Купить молоко
2. [ ] Позвонить маме
3. [ ] Сделать домашку
> done 2
Задача #2 отмечена как выполненная
> list
1. [ ] Купить молоко
2. [x] Позвонить маме
3. [ ] Сделать домашку
> remove 1
Задача #1 удалена
> list
2. [x] Позвонить маме
3. [ ] Сделать домашку
> done abc
ID должен быть числом: abc
> done 99
Задача не найдена: 99
> remove
Использование: remove <ID>
> exit
До встречи!
```

## Требования

1. Реализована функция `findIndex(tasks []Task, id int) int`, возвращающая `-1` если не нашли.
2. Команда `done <ID>`:
   - Без аргумента → подсказка по использованию
   - С нечисловым ID → ошибка
   - С несуществующим ID → «Задача не найдена»
   - С валидным ID → меняет `Done` на `true` через индекс
3. Команда `remove <ID>` с теми же проверками — удаляет задачу из слайса.
4. После `remove` в слайсе **становится на один элемент меньше**.
5. **ID удалённых задач не переиспользуются** — следующий `add` использует следующий `nextID`, а не «дырку» от удалённой.
6. Все три команды (`add`, `done`, `remove`) показывают подтверждение.
7. Все четыре команды плюс `list` и `exit` продолжают работать.

## Шаги

1. Возьми код из модуля 3
2. Импортируй `strconv`
3. Напиши функцию `findIndex(tasks []Task, id int) int`
4. Добавь два новых `case` в `switch`: `"done"` и `"remove"`
5. В каждом: проверь arg, парси через `strconv.Atoi`, найди индекс, проверь -1
6. Для `done` — изменение через индекс
7. Для `remove` — удаление через `append(tasks[:i], tasks[i+1:]...)`
8. Прогоняй сценарий из примера выше — каждую команду

## Эксперименты

1. **Удали через range:** напиши удаление неправильно — попробуй сделать его внутри `for _, t := range tasks`. Что произошло? Почему **нельзя** удалять элементы слайса прямо во время перебора через range?

2. **Удали последний элемент:** добавь 3 задачи, удали третью. Программа упала? Если нет — почему `tasks[i+1:]` не вышло за границы?

3. **Удали первый элемент:** добавь 3 задачи, удали первую. Что произошло с `tasks[:i]`, когда `i == 0`? Какой результат у `tasks[:0]`?

4. **Удали все по очереди:** добавь 3 задачи, удали `1`, потом `2`, потом `3`. Что происходит? Это подсказка к ловушке: ID не пересчитываются, поэтому удаление работает по их «старым» номерам.

5. **Используй `slices.Delete`** (если у тебя Go 1.21+). Сравни с классическим вариантом по читаемости.

6. **Добавь команду `clear`**, которая удаляет все задачи разом. Самый простой способ — `tasks = tasks[:0]` или `tasks = nil`. В чём разница между этими вариантами? Подумай про память.
