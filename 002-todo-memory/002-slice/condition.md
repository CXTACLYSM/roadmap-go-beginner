# Модуль 2. Слайс задач — условие

## Задание

Расширь программу из модуля 1 так, чтобы она работала со **списком задач**, а не с одной задачей. Список задач пока **жёстко задан в коде** — без ввода от пользователя. В модуле 3 ты сделаешь интерактивную часть.

## Что должна делать программа

```
Всего задач: 3
---
1. [ ] Купить молоко
2. [ ] Позвонить маме
3. [x] Сделать домашку
---
После пометки первой задачи как выполненной:
---
1. [x] Купить молоко
2. [ ] Позвонить маме
3. [x] Сделать домашку
```

## Требования

1. Тип `Task` и метод `String()` из модуля 1 — оставить как есть.
2. В `main` создаётся слайс `[]Task` с тремя задачами.
3. Третья задача создаётся уже как выполненная (`Done: true`).
4. Программа выводит количество задач через `len`.
5. Программа выводит **все задачи** через `for ... range`.
6. После первого вывода **первая** задача отмечается как выполненная.
7. Программа выводит список снова.
8. Между блоками вывода — разделитель `---`.

## Шаги

1. Возьми код из модуля 1 (тип `Task`, метод `String`)
2. Удали `MarkDone` — он пока не нужен (вернётся в модуле 4)
3. В `main` объяви слайс `var tasks []Task`
4. Добавь три задачи через `append` (или сразу через литерал — твой выбор)
5. Выведи `len(tasks)`
6. Сделай первый цикл `for ... range`, выведи каждую задачу через `String()`
7. Помечай первую задачу как выполненную: подумай, как это сделать **правильно**
8. Сделай второй цикл и выведи снова

## Эксперименты

1. **Попробуй пометить через range:**

   ```go
   for _, t := range tasks {
       if t.ID == 1 {
           t.Done = true
       }
   }
   ```

   Запусти. Что произошло? Почему задача не отметилась? **Это самый важный эксперимент модуля.**

2. **Сделай то же самое через индекс:**

   ```go
   for i := range tasks {
       if tasks[i].ID == 1 {
           tasks[i].Done = true
       }
   }
   ```

   Теперь работает? Почему?

3. **Забудь присваивание у append:** напиши `append(tasks, newTask)` вместо `tasks = append(tasks, newTask)`. Что изменилось? Сколько задач теперь в слайсе? Почему компилятор не предупредил?

4. **Залезь за границу:** `fmt.Println(tasks[10])`. Запусти. Прочитай сообщение паники.

5. **Посмотри на cap:** добавь `fmt.Println("len:", len(tasks), "cap:", cap(tasks))` после каждого `append`. Заметь, как они меняются. Это поможет тебе почувствовать рост слайса.

## Что показать Claude

Покажи Claude эксперимент 1 — код и результат. Спроси: «почему `t.Done = true` внутри range не работает?». Claude задаст встречный вопрос: «помнишь модуль 1, где receiver-значение был копией? Что общего между этим и `range`?». Подумай — это **одно и то же явление** в двух разных формах.

Если ты увидишь эту связь — ты сделал большой концептуальный скачок. Многие Go-разработчики годами повторяют эту ошибку, потому что не разобрались с ней в самом начале.
