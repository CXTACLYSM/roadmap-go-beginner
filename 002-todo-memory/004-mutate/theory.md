# Модуль 4. Изменение и удаление — теория

## Контекст

Сейчас твоё приложение умеет добавлять задачи и показывать список. Это половина работы. Настоящий to-do лист должен уметь:

- **Отмечать задачи как выполненные** — иначе непонятно, что уже сделано
- **Удалять задачи** — иначе список будет расти бесконечно

Обе операции требуют двух новых навыков:

1. **Найти задачу по ID** — перебрать слайс и сравнить
2. **Изменить или удалить элемент в слайсе** — это технически нетривиально

Удаление из слайса — это, наверное, самая «странная» базовая операция в Go. Если ты пришёл из других языков, ты ожидаешь чего-то вроде `tasks.delete(i)` или `tasks.remove(task)`. В Go ничего подобного нет — удаление делается комбинацией `append` и **слайсинга**. Звучит страшно, но через 5 минут станет обычным.

## Ядро 1: поиск элемента по ID

Поиск — это просто перебор:

```go
func findIndex(tasks []Task, id int) int {
    for i, t := range tasks {
        if t.ID == id {
            return i
        }
    }
    return -1
}
```

Заметь несколько важных моментов:

1. Возвращаем **индекс**, а не саму задачу. Через индекс можно и прочитать, и изменить, и удалить — а через копию задачи ничего не сделаешь.
2. Если не нашли — возвращаем `-1`. Это **сигнальное значение** — индекс, которого не может быть в нормальном слайсе. Вызывающая сторона должна это проверить.
3. `range` здесь безопасен — мы только читаем `t.ID`, не пытаемся менять `t`.

Альтернатива — возвращать `(int, error)`, если «не нашли» — это серьёзная ошибка, требующая особой обработки. Для нашей задачи `-1` достаточно: «не нашли — пользователь ошибся, скажем ему вежливо».

## Ядро 2: изменение элемента слайса

Помнишь правило из модуля 2? **Изменение через range не работает, изменение через индекс — работает.**

```go
i := findIndex(tasks, id)
if i == -1 {
    fmt.Println("Задача не найдена:", id)
    return
}
tasks[i].Done = true   // <-- через индекс, изменение видно
```

Это **прямой доступ** к полю элемента слайса. Никаких копий, изменение сохраняется.

## Ядро 3: удаление элемента из слайса

Самая нетривиальная часть. В Go нет встроенной функции `delete` для слайсов (есть для мап, но это другое). Удаление делается через **слайсинг** и `append`.

Идиоматичный паттерн:

```go
tasks = append(tasks[:i], tasks[i+1:]...)
```

Звучит как заклинание. Давай разберём.

### Что такое `tasks[:i]` и `tasks[i+1:]`

Это **операции слайсинга** — создание нового слайса, который смотрит на **часть** существующего.

```go
tasks := []Task{a, b, c, d, e}
//        0  1  2  3  4

tasks[:2]   // [a, b]      — от начала (включительно) до индекса 2 (НЕ включая)
tasks[2:]   // [c, d, e]   — от индекса 2 (включительно) до конца
tasks[1:4]  // [b, c, d]   — от индекса 1 до индекса 4 (НЕ включая)
tasks[:]    // [a, b, c, d, e]  — весь слайс
```

Запомни правило: **левая граница включается, правая — нет**. Это как с диапазонами в большинстве языков, и это удобно, потому что `tasks[:i]` и `tasks[i:]` вместе дают исходный слайс без дублирования и без пропусков.

### Применяем к удалению

Пусть ты хочешь удалить элемент с индексом `2` (т.е. `c`).

- `tasks[:2]` → `[a, b]` — всё **до** удаляемого
- `tasks[2+1:]` → `tasks[3:]` → `[d, e]` — всё **после** удаляемого
- `append(tasks[:2], tasks[3:]...)` → `[a, b, d, e]`

Готово. `c` исчез.

Заметь `...` в конце второго аргумента — это **распаковка** слайса в отдельные элементы. Без них Go подумает, что ты пытаешься добавить **сам слайс** в качестве одного элемента, и это не скомпилируется (нельзя добавить `[]Task` в `[]Task` как один элемент).

И не забудь **присваивание обратно**: `tasks = append(...)`. Помнишь, `append` возвращает новый слайс.

## Ловушка: что происходит с памятью при удалении

Если ты любопытный — вот что на самом деле происходит. Это не для зубрёжки, а для интуиции.

Слайс `tasks[:2]` указывает на **тот же массив в памяти**, что и `tasks`. Когда `append` копирует туда `tasks[3:]`, он перезаписывает старые значения **поверх**.

```
Было:    [a][b][c][d][e]
              ↓
После:   [a][b][d][e][?]   ← вместо c теперь d, вместо d теперь e, последний слот «мусор»
```

`append` возвращает слайс с длиной 4. Старый слот (индекс 4) формально «не виден», но память занята. Это нормально и не страшно — если в нём лежал указатель, и ты не хочешь, чтобы он мешал сборщику мусора, его можно явно занулить. Но для нашей задачи это излишне.

Если хочешь видеть это руками — после удаления выведи `len(tasks)` и `cap(tasks)`. Длина уменьшится, а ёмкость — нет.

## Альтернативный способ удаления (Go 1.21+)

В современных версиях Go появилась функция `slices.Delete` из пакета `slices`:

```go
import "slices"

tasks = slices.Delete(tasks, i, i+1)
```

Делает то же самое, но читается яснее. **Я рекомендую использовать её**, если у тебя Go 1.21 или новее (проверь через `go version`). Но **знать классический способ** через `append` — обязательно: ты будешь видеть его в чужом коде постоянно.

## Полный пример: команды done и remove

```go
case "done":
    if arg == "" {
        fmt.Println("Использование: done <ID>")
        continue
    }
    id, err := strconv.Atoi(arg)
    if err != nil {
        fmt.Println("ID должен быть числом:", arg)
        continue
    }
    i := findIndex(tasks, id)
    if i == -1 {
        fmt.Println("Задача не найдена:", id)
        continue
    }
    tasks[i].Done = true
    fmt.Printf("Задача #%d отмечена как выполненная\n", id)

case "remove":
    if arg == "" {
        fmt.Println("Использование: remove <ID>")
        continue
    }
    id, err := strconv.Atoi(arg)
    if err != nil {
        fmt.Println("ID должен быть числом:", arg)
        continue
    }
    i := findIndex(tasks, id)
    if i == -1 {
        fmt.Println("Задача не найдена:", id)
        continue
    }
    tasks = append(tasks[:i], tasks[i+1:]...)
    fmt.Printf("Задача #%d удалена\n", id)
```

Заметь, как много кода **повторяется**: проверка пустого аргумента, парсинг ID, поиск задачи, проверка `-1`. Это сигнал к рефакторингу — мы займёмся им в модуле 5.

## strconv.Atoi для парсинга ID

`strconv.Atoi(s)` — это «**A**SCII **to** **i**nteger». Превращает строку в `int`:

```go
id, err := strconv.Atoi("42")  // id = 42, err = nil
id, err := strconv.Atoi("abc") // id = 0, err = "invalid syntax: abc"
```

Похоже на `ParseFloat` из проекта 1, только для целых.
