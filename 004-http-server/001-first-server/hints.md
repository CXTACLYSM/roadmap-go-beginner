## Уровень 2: про сигнатуру handler-а

Эта сигнатура — **константа Go**, её надо запомнить:

```go
func имя(w http.ResponseWriter, r *http.Request)
```

- **`w http.ResponseWriter`** — без указателя (это интерфейс, который под капотом реализован как указатель)
- **`r *http.Request`** — с указателем

Если ты напишешь `r http.Request` (без `*`) — компилятор скажет, что сигнатура неправильная.

## Уровень 3: про fmt.Fprintln с ResponseWriter

`ResponseWriter` реализует интерфейс `io.Writer`, поэтому в него можно писать **всеми** функциями `fmt`, у которых есть форма `F...`:

```go
fmt.Fprintln(w, "Hello, world!")        // как Println, но в Writer
fmt.Fprintf(w, "ID: %d\n", task.ID)    // как Printf
fmt.Fprint(w, "no newline")             // как Print
```

Альтернатива — использовать `w.Write([]byte("..."))` напрямую, но это менее удобно. Для строк всегда `Fprintln`/`Fprintf`.

## Уровень 4: про ListenAndServe

```go
err := http.ListenAndServe(":8080", nil)
if err != nil {
    fmt.Println("Ошибка сервера:", err)
}
```

Несколько важных моментов:

- **`":8080"`** — двоеточие в начале, потом порт. Без двоеточия не сработает.
- **`nil`** во втором аргументе — означает «использовать глобальный мультиплексор». Если ты регистрировал обработчики через `http.HandleFunc`, они оказались именно в глобальном — `nil` найдёт их.
- **Это блокирующий вызов.** Программа застревает в `ListenAndServe` и не выходит из него, пока сервер не упадёт. Поэтому он идёт **в самом конце** `main`.
- **Возвращает ошибку**, если сервер не смог запуститься (например, порт занят) или упал. В нормальной работе никогда не возвращает `nil`.

## Уровень 5: про доступ к данным запроса

Из `r *http.Request` можно достать много полезного. Для модуля 1 тебе нужно:

```go
r.Method                    // "GET", "POST" и т.д.
r.URL.Path                  // "/info", "/hello"
r.Header.Get("User-Agent")  // значение конкретного заголовка
```

`r.Header` — это мапа, метод `Get` возвращает первое значение или пустую строку, если заголовка нет.

## Уровень 6: про порт занят

Если ты увидел такую ошибку:

```
listen tcp :8080: bind: address already in use
```

— это значит, что **на этом порту уже кто-то слушает**. Варианты:

1. У тебя в другом терминале запущен старый сервер — останови его (Ctrl+C там).
2. Ты дважды запустил программу — закрой одну.
3. Этот порт занял другой процесс. Найди его и убей:
   ```bash
   lsof -i :8080      # увидеть, кто занял (Linux/macOS)
   kill <PID>          # убить процесс
   ```

Или просто возьми другой порт — `8081`, `8082`, любой свободный.

## Уровень 7: про порт 80

Если ты попробовал слушать на порту 80:

```
listen tcp :80: bind: permission denied
```

— это нормально. Порты ниже 1024 — **привилегированные**, для них нужны права root. На Linux/macOS в продакшене обычно стартуют сервер от root, потом сразу понижают права. Или используют **обратный прокси** (nginx) на порту 80, который проксирует на твой Go-сервер на порту 8080. Подробнее — в Проекте 7.

## Уровень 8: про порядок WriteHeader и Write

Если в твоём коде есть оба вызова — порядок **критичен**:

```go
// ПРАВИЛЬНО
w.WriteHeader(http.StatusOK)
fmt.Fprintln(w, "Hello")

// НЕПРАВИЛЬНО — первый Write автоматически отправит 200, второй WriteHeader будет проигнорирован
fmt.Fprintln(w, "Hello")
w.WriteHeader(http.StatusInternalServerError)
```

В неправильном случае Go выведет предупреждение в логи и пошлёт клиенту 200, а не 500.

Логика такая: `WriteHeader` отправляет **стартовую строку и заголовки**. После этого тело уже идёт. После начала тела заголовки и статус **изменить нельзя** — они уже улетели. Поэтому **сначала** все заголовки и статус, **потом** тело.

В модуле 1 это не критично (мы используем только `Fprintln`), но в модулях 2-4 будет важно.

## Если совсем тупик

Скажи Claude конкретно: «модуль 1 проекта 4, [конкретная проблема]». Покажи код и сообщение об ошибке. Самые частые ситуации:

- «`undefined: http`» → забыл импортировать `net/http`
- «`not enough arguments in call to http.HandleFunc`» → забыл второй аргумент-handler
- «`listen tcp :8080: bind: address already in use`** → см. уровень 6
- «Сервер запустился, но curl не отвечает» → возможно, сервер слушает на `localhost:8080`, а ты обращаешься на `127.0.0.1:8080`. Они должны быть эквивалентны, но проверь, что нет опечаток в адресе
