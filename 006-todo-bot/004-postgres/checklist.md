## Подготовка к модулю 5

В модуле 5 ты добавишь **состояние диалога**: возможность вводить `/add` без аргумента, и бот будет ждать текст следующего сообщения.

Главные новые мысли:

- **Finite State Machine (FSM)** — простейшая форма управления состоянием диалога
- **Состояние per user** — у каждого пользователя своё «текущее ожидание»
- **Маршрутизация по состоянию** — если у пользователя есть состояние, обработать сообщение **по нему**, а не как команду

Это превратит бота из «формы для ввода команд» в **полноценный диалог**.
