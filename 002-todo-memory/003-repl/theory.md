# Модуль 3. REPL — цикл команд — теория

## Контекст

В предыдущих модулях твои программы были **однократными**: запустился, что-то сделал, завершился. Это нормально для калькулятора или скрипта, но не годится для интерактивного приложения.

Настоящий to-do лист должен **жить** во времени: запуститься, ждать команд, обрабатывать каждую, запоминать состояние между командами, и завершаться только когда пользователь явно скажет.

Этот шаблон называется **REPL** — **R**ead, **E**val, **P**rint, **L**oop:

1. **Read** — прочитать ввод от пользователя
2. **Eval** — выполнить команду (изменить состояние, что-то посчитать)
3. **Print** — вывести результат
4. **Loop** — вернуться к шагу 1

Этот же шаблон лежит в основе командной строки (bash), в основе Python/Node.js REPL, в основе любой интерактивной игры. И, что важно для тебя как будущего бэкендера — **HTTP-сервер устроен по тому же принципу**: бесконечный цикл, в котором каждая итерация обрабатывает один запрос. Так что REPL — это не игрушка, это репетиция настоящих серверов.

## Ядро: бесконечный цикл

В Go только один цикл — `for`. Бесконечный цикл выглядит так:

```go
for {
    // что-то делать
}
```

Без условия — значит, выполняется вечно. Прервать его можно через:

- `break` — выйти из цикла
- `return` — выйти из функции (а вместе с ней и из цикла)
- `os.Exit(...)` — завершить программу совсем

Для REPL обычно используют `break` или `return`, потому что хочется корректно дойти до конца программы и, например, попрощаться с пользователем.

## Чтение строки от пользователя: bufio.Scanner

Помнишь `fmt.Scan` из проекта 1? Он удобен для чтения **слова** или **числа**, но плох для нашей задачи. Почему?

Команда вроде `add Купить молоко` содержит **пробелы**. `fmt.Scan` останавливается на первом пробеле — тебе пришлось бы вызывать его несколько раз и склеивать, что хрупко и некрасиво.

Для чтения **целых строк** в Go есть пакет `bufio` (от **buf**fered **i**nput/**o**utput) и тип `Scanner`:

```go
import (
    "bufio"
    "os"
)

scanner := bufio.NewScanner(os.Stdin)

for scanner.Scan() {
    line := scanner.Text()
    // обработать line
}
```

Разберём по частям:

- `bufio.NewScanner(os.Stdin)` — создаёт сканер, который читает из стандартного ввода
- `scanner.Scan()` — читает **одну строку** до символа перевода `\n`. Возвращает `true`, если строка прочитана, и `false`, если ввод закончился (EOF) или произошла ошибка
- `scanner.Text()` — возвращает прочитанную строку **без** перевода в конце

Конструкция `for scanner.Scan() { ... }` — идиома Go, означающая «читать строку за строкой, пока есть что читать».

## Зачем приглашение и как его сделать

Чтобы пользователь понимал, что программа ждёт ввода, печатают **приглашение** (prompt) — обычно `>` или `> `:

```go
for {
    fmt.Print("> ")              // приглашение
    if !scanner.Scan() {         // читаем строку
        break                    // ввод закончился — выходим
    }
    line := scanner.Text()
    // обработать line
}
```

Заметь две вещи:

1. **`fmt.Print`** (без `ln`) — без перевода строки. Курсор остаётся на той же строке, чтобы ввод пользователя выглядел рядом с приглашением.
2. Проверка `if !scanner.Scan()` — если `Scan` вернул `false` (например, пользователь нажал Ctrl+D на Linux/macOS, что означает «конец ввода») — мы выходим из цикла.

## Парсинг команды

Когда у тебя есть строка `"add Купить молоко"`, тебе нужно отделить **команду** (`add`) от **аргументов** (`Купить молоко`). Самый простой способ — пакет `strings` и функция `SplitN`:

```go
import "strings"

parts := strings.SplitN(line, " ", 2)
```

`SplitN(s, sep, n)` разбивает строку `s` по разделителю `sep`, но **максимум на `n` частей**. Это важно: если ты используешь обычный `Split`, строка `"add Купить молоко"` превратится в `["add", "Купить", "молоко"]` — три части. А с `SplitN(line, " ", 2)` ты получишь `["add", "Купить молоко"]` — ровно две: команда и весь остаток.

После этого:

```go
command := parts[0]
var arg string
if len(parts) > 1 {
    arg = parts[1]
}
```

Проверка `len(parts) > 1` нужна для команд **без аргументов** (`list`, `exit`) — у них в строке только одно «слово», и `parts[1]` не существует.

## Выбор команды: switch

Для разбора команды отлично подходит `switch`:

```go
switch command {
case "add":
    // добавить задачу с текстом arg
case "list":
    // вывести все задачи
case "exit":
    fmt.Println("До встречи!")
    return
default:
    fmt.Println("Неизвестная команда:", command)
}
```

`return` в `case "exit"` корректно выходит из `main` (если REPL — внутри `main`) или из функции, в которой живёт цикл.

## Состояние между итерациями

Самый важный концептуальный момент модуля.

**Слайс задач объявляется ДО цикла**, а не внутри него:

```go
func main() {
    var tasks []Task          // <-- здесь, до цикла
    scanner := bufio.NewScanner(os.Stdin)

    for {
        fmt.Print("> ")
        if !scanner.Scan() {
            break
        }
        line := scanner.Text()
        // ... парсинг команды ...

        switch command {
        case "add":
            tasks = append(tasks, Task{...})  // <-- меняем внешнюю переменную
        case "list":
            for _, t := range tasks {         // <-- читаем внешнюю переменную
                fmt.Println(t.String())
            }
        }
    }
}
```

Если ты объявишь `var tasks []Task` **внутри** цикла — каждая итерация будет создавать новый пустой слайс, и состояние не сохранится. Это самый частый «странный баг» новичков на этом этапе.

## Полный скелет REPL

```go
package main

import (
    "bufio"
    "fmt"
    "os"
    "strings"
)

type Task struct {
    ID    int
    Title string
    Done  bool
}

func (t Task) String() string {
    status := "[ ]"
    if t.Done {
        status = "[x]"
    }
    return fmt.Sprintf("%d. %s %s", t.ID, status, t.Title)
}

func main() {
    var tasks []Task
    nextID := 1
    scanner := bufio.NewScanner(os.Stdin)

    fmt.Println("To-Do приложение. Команды: add <текст>, list, exit")

    for {
        fmt.Print("> ")
        if !scanner.Scan() {
            break
        }
        line := strings.TrimSpace(scanner.Text())
        if line == "" {
            continue
        }

        parts := strings.SplitN(line, " ", 2)
        command := parts[0]
        var arg string
        if len(parts) > 1 {
            arg = parts[1]
        }

        switch command {
        case "add":
            if arg == "" {
                fmt.Println("Использование: add <текст задачи>")
                continue
            }
            tasks = append(tasks, Task{ID: nextID, Title: arg})
            fmt.Printf("Задача добавлена: #%d %s\n", nextID, arg)
            nextID++
        case "list":
            if len(tasks) == 0 {
                fmt.Println("Список пуст")
                continue
            }
            for _, t := range tasks {
                fmt.Println(t.String())
            }
        case "exit":
            fmt.Println("До встречи!")
            return
        default:
            fmt.Println("Неизвестная команда:", command)
        }
    }
}
```

Несколько новых деталей:

- **`strings.TrimSpace(s)`** — убирает пробелы в начале и конце строки. Полезно, чтобы случайные пробелы не ломали парсинг.
- **`continue`** — переход к следующей итерации цикла, минуя оставшийся код. Удобно для «эта команда не подходит, пропускаем дальше».
- **`nextID`** — счётчик ID. Тоже объявлен **до цикла**, чтобы сохранять значение между итерациями.
