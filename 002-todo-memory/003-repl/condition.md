# Модуль 3. REPL — цикл команд — условие

## Задание

Превратить программу в **интерактивное приложение**, которое после запуска ждёт команд от пользователя, обрабатывает их и продолжает работать, пока пользователь не введёт `exit`.

## Что должна делать программа

```
$ go run main.go
To-Do приложение. Команды: add <текст>, list, exit
> add Купить молоко
Задача добавлена: #1 Купить молоко
> add Позвонить маме
Задача добавлена: #2 Позвонить маме
> list
1. [ ] Купить молоко
2. [ ] Позвонить маме
> add Сделать домашку
Задача добавлена: #3 Сделать домашку
> list
1. [ ] Купить молоко
2. [ ] Позвонить маме
3. [ ] Сделать домашку
> exit
До встречи!
```

## Требования

1. Используется `bufio.Scanner` для чтения строк, **не** `fmt.Scan`.
2. Бесконечный цикл `for { ... }`.
3. Перед каждым ожиданием ввода выводится приглашение `> `.
4. Поддерживаются **три команды**: `add <текст>`, `list`, `exit`.
5. После `add` в слайс добавляется новая задача с уникальным `ID` (1, 2, 3, ...).
6. ID **не сбрасывается** после удаления задач (этого пока нет, но привыкай).
7. После `add` выводится подтверждение с номером и текстом задачи.
8. `list` без задач выводит «Список пуст».
9. `exit` выводит «До встречи!» и завершает программу через `return`.
10. Неизвестная команда — сообщение «Неизвестная команда: ...», программа продолжает работать.
11. Пустой ввод (просто Enter) — игнорируется, программа продолжает работать.
12. Слайс задач и счётчик ID объявлены **до** цикла.
13. Команды `done` и `remove` пока **не реализуем** — это модуль 4.

## Шаги

1. Возьми код из модуля 2
2. Удали жёстко прописанные задачи — слайс должен быть пустым в начале
3. Добавь импорты: `bufio`, `os`, `strings`
4. Создай `bufio.NewScanner(os.Stdin)` перед циклом
5. Объяви `nextID := 1` для счётчика
6. Сделай бесконечный цикл `for { ... }`
7. Внутри: приглашение → `scanner.Scan()` → `scanner.Text()` → парсинг
8. `strings.TrimSpace` для очистки строки
9. Если пустая строка — `continue`
10. `strings.SplitN(line, " ", 2)` — разбить на команду и аргумент
11. `switch` по команде: add, list, exit, default
12. Прогоняй всё руками: добавь несколько задач, посмотри список, выйди

## Эксперименты

1. **Объяви `var tasks []Task` ВНУТРИ цикла** (на первой строке после `for {`). Запусти, добавь задачу, сделай list. Что произошло? Почему? Верни как было.

2. **Используй `Split` вместо `SplitN`:**
   ```go
   parts := strings.Split(line, " ")
   ```
   Попробуй добавить задачу с длинным текстом: `add Купить молоко и хлеб`. Что увидишь? Почему?

3. **Забудь `TrimSpace`.** Введи команду с пробелами в начале или конце: `  list  `. Что произошло? Почему `switch` не сработал?

4. **Нажми Ctrl+D** (на Linux/macOS) или Ctrl+Z затем Enter (на Windows) вместо `exit`. Программа корректно завершилась? Это и есть «ввод закончился» — `scanner.Scan()` вернул `false`.

5. **Введи `exit ` с пробелом в конце** (без TrimSpace). Сработало? Почему `switch case "exit"` не подходит к строке `"exit "`?

6. **Добавь команду `count`**, которая выводит количество задач (просто `len(tasks)`). Сколько строк ты добавил? В скольких местах?

## Что показать Claude

Покажи Claude эксперимент 1 (когда `var tasks []Task` объявлен внутри цикла). Спроси: «почему задачи не сохраняются между командами?». Это ключевой момент модуля — понимание, что **состояние живёт в переменной, объявленной до цикла**. Без этого понимания невозможно писать никакие сервера.
