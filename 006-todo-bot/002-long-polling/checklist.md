# Модуль 2. Long polling и echo-бот — чек-лист готовности

## Поведенческие маркеры

- [ ] Расширен `TelegramClient` методом `GetUpdates(offset int64) ([]Update, error)`
- [ ] Расширен `TelegramClient` методом `SendMessage(chatID int64, text string) error`
- [ ] Объявлены типы `Update`, `Message`, `Chat` с правильными JSON-тегами
- [ ] **`http.Client` имеет таймаут больше 30 секунд** (или 0)
- [ ] `GetUpdates` использует параметр `timeout=30` в URL
- [ ] `SendMessage` использует HTTP POST с JSON-телом и Content-Type
- [ ] **`defer resp.Body.Close()`** в каждом методе HTTP
- [ ] В `main` объявлен `var offset int64 = 0` **до** цикла
- [ ] Бесконечный цикл `for { ... }` с `GetUpdates(offset)`
- [ ] При ошибке `GetUpdates` — лог + `time.Sleep(5 * time.Second)` + `continue`
- [ ] Цикл по обновлениям проверяет `upd.Message != nil`
- [ ] Цикл по обновлениям проверяет `upd.Message.Text != ""`
- [ ] **`offset` обновляется** после обработки: `offset = upd.UpdateID + 1`
- [ ] Бот логирует каждое полученное сообщение
- [ ] Бот отвечает «Ты сказал: <текст>»
- [ ] При ошибке `SendMessage` — лог, **не падение**
- [ ] **Бот реально работает в Telegram** — пишешь, он отвечает
- [ ] После перезапуска бот **не отвечает повторно** на старые сообщения (offset)

## Подготовка к модулю 3

Сейчас твой бот **эхо-бот** — отвечает теми же словами. В модуле 3 ты добавишь **команды**:

- `/start` — приветствие и список команд
- `/help` — справка
- `/list` — показать задачи
- `/add <текст>` — добавить задачу
- `/done <id>` — отметить выполненной
- `/delete <id>` — удалить
- Любой другой текст — ответ «не понимаю, /help»

Главные новые мысли модуля 3:

- **Парсинг команды** — отделить `/add` от аргумента «купить молоко»
- **Хранилище в памяти** — пока без БД, простая `map[chatID][]Task`
- **Per-user state** — у каждого пользователя свой список задач
- **Свитч по командам** — паттерн, который ты уже видел в Проекте 2 (REPL)

В модуле 3 у тебя получится **функционально полный** to-do бот, но без БД. БД будет в модуле 4.
