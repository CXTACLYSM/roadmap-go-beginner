# Модуль 4. Разделение кода по файлам — подсказки

> ⚠️ Открывай только после 30 минут самостоятельной борьбы.

## Уровень 1: наводящие вопросы

1. Какое объявление пакета должно быть в начале **каждого** файла твоего проекта?
2. Нужно ли импортировать `task.go` из `main.go`, чтобы использовать `Task`?
3. Какая команда запустит проект из нескольких файлов?
4. По какому критерию ты решаешь, в какой файл положить функцию?

## Уровень 2: про `package main` в каждом файле

Каждый `.go` файл должен начинаться с объявления пакета:

```go
package main
```

**Все файлы в одной директории должны иметь одно и то же имя пакета.** Если в одном файле `package main`, а в другом `package todo` — компилятор скажет:

```
found packages main (main.go) and todo (task.go)
```

Решение: исправь несовпадающие.

## Уровень 3: про импорты

Импорты в каждом файле — **только то, что использует этот файл**. Не общий список на весь проект.

Например, `task.go` использует только `fmt`:

```go
package main

import "fmt"

type Task struct { ... }
func (t Task) String() string { ... }
```

А `tasklist.go` использует много чего:

```go
package main

import (
    "encoding/json"
    "errors"
    "fmt"
    "os"
)
```

Если у тебя несколько импортов — оборачивай в скобки:

```go
import (
    "fmt"
    "os"
)
```

— это идиоматичная Go-форма для нескольких импортов.

## Уровень 4: про неиспользуемые импорты

Если ты импортировал пакет, но не используешь его — компилятор **ошибётся**:

```
imported and not used: "encoding/json"
```

Это **нарочно**. В других языках разрешено иметь висячие импорты, но в Go это считается мусором. Удали ненужный импорт.

Это особенно часто случается при рефакторинге. Ты переносишь функцию `Save` из `main.go` в `tasklist.go`, и `main.go` больше не использует `encoding/json`. Удали этот импорт оттуда.

## Уровень 5: про go run .

```bash
# Старая форма (только для одного файла)
go run main.go

# Новая форма (для пакета из любого числа файлов)
go run .
```

Точка означает «текущая директория». Go находит все `.go` файлы (кроме файлов с `_test.go` — это тесты), собирает их в один пакет и запускает.

Если у тебя в той же папке есть `package main` и `func main()` — это исполняемая программа. Если нет `func main()` — это пакет-библиотека, и `go run .` выдаст ошибку «no main function».

## Уровень 6: про порядок переноса

Самый безопасный порядок при разделении:

1. **Сначала простое** — типы и их методы, у которых мало зависимостей. Например, `Task` и `String()`. Перенеси, проверь, что компилируется.
2. **Потом среднее** — типы с большей логикой. `TaskList` и его методы. Перенеси, проверь.
3. **Потом верхнее** — обработчики и helper'ы. Они зависят от всего остального.
4. **В конце** — что осталось в `main.go`. Это должен быть короткий файл с REPL и helper-ами.

После **каждого** шага запускай `go run .` и проверяй, что работает. Если сломалось — ты знаешь, **что именно** ты только что сделал.

## Уровень 7: про чистку импортов в main.go

Самая распространённая ошибка: ты перенёс код, но забыл почистить импорты в исходном файле. Например, перенёс `Save`/`Load` в `tasklist.go`, а в `main.go` остался импорт `encoding/json`, который теперь не нужен.

Лекарство:

1. Запусти `go build .` или `go vet .` — компилятор покажет неиспользуемые импорты
2. Удали их вручную
3. Или используй `goimports -w .` — он сам почистит

VSCode с Go-расширением делает это автоматически при сохранении файла. Если у тебя так настроено — проблема не возникает.

## Уровень 8: про gofmt

`gofmt` — встроенный форматёр Go, идёт в комплекте с языком. Команды:

```bash
gofmt -l .       # показать файлы, которые отформатированы неправильно (только имена)
gofmt -d .       # показать diff между текущим и правильным форматированием
gofmt -w .       # переформатировать файлы на месте
```

В Go-проектах **принято** держать код в формате `gofmt`. Это убирает все споры «как форматировать» — есть один правильный способ, и он встроен в язык.

`goimports` — расширенная версия, которая также управляет импортами. Рекомендую установить:

```bash
go install golang.org/x/tools/cmd/goimports@latest
```

И использовать `goimports -w .` вместо `gofmt -w .`.

## Если совсем тупик

Скажи Claude конкретно: «модуль 4, не работает [конкретная штука]». Покажи структуру файлов и сообщение об ошибке. Самые частые ситуации:

- **`undefined: Task`** → у файлов разные `package`, или ты забыл сохранить файл
- **`imported and not used`** → нужно почистить импорты в файле, откуда унёс код
- **`go run main.go` не работает** → используй `go run .`
- **`syntax error: non-declaration statement outside function body`** → в файле есть код, который должен быть внутри функции (например, забыл обернуть в `func`)
