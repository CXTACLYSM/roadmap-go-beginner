# Модуль 5. Декомпозиция и обработка ошибок — условие

## Задание

**Это рефакторинг.** Ты не добавляешь нового функционала — ты переписываешь существующий код так, чтобы он стал чище. Поведение программы для пользователя должно остаться **точно таким же**.

## Что нужно сделать

1. Создать тип `TaskList` (структура с полями `tasks []Task` и `nextID int`)
2. Перенести логику работы со списком в **методы** этого типа: `Add`, `MarkDone`, `Remove`, `findIndex`, `All`
3. Методы `MarkDone` и `Remove` возвращают `error`, а не выводят сообщения сами
4. Создать функции-обработчики для каждой команды: `handleAdd`, `handleList`, `handleDone`, `handleRemove`
5. Создать функцию `parseCommand(line string) (string, string)` для парсинга
6. Создать функцию `parseID(arg, command string) (int, error)` — она используется и в `done`, и в `remove`
7. `main` после рефакторинга должен стать **тонким**: только цикл, switch и вызовы handle-функций

## Критерии «хорошего» main

После рефакторинга функция `main` должна:

- Помещаться целиком на одном экране (~30 строк или меньше)
- Не содержать ни одной строчки **бизнес-логики** (никаких `tasks[i].Done = true`, никаких `append(tasks[:i]...)`)
- Содержать только: создание `TaskList`, создание сканера, цикл, switch, вызовы handle-функций

## Требования к поведению (без изменений)

Все сценарии из модуля 4 должны работать **точно так же**:

- `add <текст>` добавляет, выводит подтверждение
- `list` показывает все задачи или «Список пуст»
- `done <ID>` отмечает или сообщает об ошибке
- `remove <ID>` удаляет или сообщает об ошибке
- Все ошибочные случаи (без аргумента, с буквами, с несуществующим ID) обрабатываются как раньше
- `exit` корректно завершает
- Неизвестная команда — сообщение, программа продолжает

## Шаги (рекомендуемый порядок)

Рефакторинг — деликатная операция. Делай маленькими шагами, и **после каждого шага запускай программу**, чтобы убедиться, что ничего не сломалось.

1. **Создай тип `TaskList`** в файле. Пока не используй — просто объяви.
2. **Перенеси `findIndex`** в метод. Старую функцию пока оставь — заменишь потом.
3. **Перенеси `Add`** в метод `TaskList.Add(title string) Task`. В `main` пока используй старый код.
4. В `main` создай экземпляр `list := &TaskList{nextID: 1}`. Переключи только команду `add` на `list.Add(arg)`. Запусти, проверь.
5. **Перенеси `MarkDone`** как метод, возвращающий `error`. Переключи команду `done`. Проверь.
6. **Перенеси `Remove`** аналогично. Проверь.
7. **Перенеси `list`** через метод `All()`. Проверь.
8. Удали старый код и переменные, которые больше не нужны.
9. Теперь **вынеси handle-функции**. Команда за командой. После каждого выноса — проверка.
10. Вынеси `parseCommand`.
11. Вынеси `parseID`.
12. Финальная проверка — прогони все сценарии.
