# Модуль 4. Разделение кода по файлам — условие

## Задание

**Это рефакторинг.** Возьми весь код Проекта 3 (после модуля 3) и разложи его на четыре файла: `main.go`, `task.go`, `tasklist.go`, `handlers.go`. Поведение программы должно остаться **точно таким же**.

## Структура после рефакторинга

```
app/
├── go.mod
├── main.go         ← REPL, helper'ы
├── task.go         ← тип Task и его методы
├── tasklist.go     ← тип TaskList и все его методы
├── handlers.go     ← parseCommand, parseID, handleXxx
└── tasks.json      ← данные (создаётся в рантайме)
```

## Требования

1. **Все файлы — один пакет `package main`** (никаких отдельных пакетов).
2. **`task.go`** содержит:
   - Тип `Task`
   - Метод `String()`
   - Импорт `fmt`
3. **`tasklist.go`** содержит:
   - Тип `TaskList`
   - Все методы: `Add`, `MarkDone`, `Remove`, `findIndex`, `All`, `Save`, `Load`
   - Импорты: `encoding/json`, `errors`, `fmt`, `os`
4. **`handlers.go`** содержит:
   - `parseCommand`, `parseID`
   - `handleAdd`, `handleList`, `handleDone`, `handleRemove`
   - Импорты: `fmt`, `strconv`, `strings`
5. **`main.go`** содержит:
   - Константу `dataFile`
   - Функцию `main` с REPL-циклом
   - Helper `saveOrWarn`
   - Импорты: `bufio`, `fmt`, `os`, `strings`
6. Каждый файл импортирует **только то, что ему нужно**.
7. Программа запускается через `go run .` (не `go run main.go`).
8. Все команды и сценарии из модуля 3 продолжают работать.
9. Код проходит `gofmt -w .` без изменений (то есть отформатирован правильно).

## Шаги

**Делай маленькими шагами!** Это рефакторинг — после каждого шага запускай программу.

1. Скопируй папку `app/` из модуля 3 в `app-multifile/` (чтобы не сломать рабочий вариант)
2. Создай пустой файл `task.go` с `package main`
3. Перенеси туда тип `Task` и метод `String()`
4. Из `main.go` удали этот код
5. Запусти `go run .` — должно работать
6. Создай `tasklist.go` с `package main`
7. Перенеси туда тип `TaskList` и **все** его методы (включая `Save`, `Load`)
8. Не забудь импорты — `encoding/json`, `errors`, `fmt`, `os`
9. Из `main.go` удали этот код, **проверь и почисти импорты в `main.go`**
10. Запусти — должно работать
11. Создай `handlers.go`
12. Перенеси туда `parseCommand`, `parseID`, все `handleXxx`
13. Не забудь импорты
14. Из `main.go` удали этот код, почисти импорты
15. Запусти `go run .` — должно работать
16. Прогон финального сценария: создание задач, exit, перезапуск, проверка

## Эксперименты

1. **Запусти `go run main.go`** (старая форма). Что произошло? Почему?

2. **Удали импорт `fmt` из `main.go`** (если он там есть). Что говорит компилятор? Это работа `go vet`-стиля строгости — ты обязан явно импортировать всё, что используешь, и не имеешь права импортировать лишнее.

3. **Запусти `gofmt -w .`** в папке проекта. Что-то изменилось? Если да — посмотри, что именно. Это автоматический форматёр приводит код к стандартному виду.

4. **Запусти `gofmt -d .`** (с ключом `-d` — diff). Покажет, что бы он изменил, не меняя файлы. Полезно в скриптах CI.

5. **Перенеси `parseCommand` обратно в `main.go`.** Запусти. Работает? Конечно — внутри одного пакета все файлы видят друг друга. Но логически это **неправильное место** для функции парсинга команды. Верни в `handlers.go`.

6. **Создай файл `_unused.go`** с произвольным `package main` и одной неиспользуемой функцией:
   ```go
   package main

   func unused() {}
   ```
   Запусти. Работает? Да — Go позволяет иметь функции, которые никто не вызывает (в отличие от неиспользуемых **переменных и импортов**). Удали файл.

7. **Создай файл `task_test.go`** с одним пустым тестом:
   ```go
   package main

   import "testing"

   func TestNothing(t *testing.T) {}
   ```
   Запусти `go test .` — должно вывести `ok`. Это твоё первое прикосновение к тестам в Go. Подробнее — в Проекте 4.
