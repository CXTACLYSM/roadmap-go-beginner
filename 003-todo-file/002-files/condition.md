# Модуль 2. Работа с файлами — условие

## Задание

Написать **отдельную программу**, которая демонстрирует чтение и запись JSON-данных в файл, включая корректную обработку случая «файла нет». Это снова **полигон**, не интеграция в TaskList — она будет в модуле 3.

## Что должна делать программа

```
$ go run .
Сохраняем 3 задачи в test.json...
Сохранено
Читаем test.json...
Загружено 3 задач:
  ID=1 Title="Купить молоко" Done=false
  ID=2 Title="Позвонить маме" Done=true
  ID=3 Title="Сделать домашку" Done=false
Пытаемся загрузить nonexistent.json...
Файла нет — это нормально, возвращаем пустой список
Загружено 0 задач

$ cat test.json
[
  {
    "id": 1,
    "title": "Купить молоко",
    "done": false
  },
  ...
]
```

## Требования

1. Создай папку `practice-files/` с собственным `go.mod`.
2. Тип `Task` с JSON-тегами (можешь скопировать из модуля 1).
3. Функция `saveTasks(filename string, tasks []Task) error`:
   - Сериализует через `MarshalIndent` с двумя пробелами
   - Записывает через `os.WriteFile` с правами `0644`
   - Возвращает обёрнутую ошибку через `fmt.Errorf("...: %w", err)` с понятным контекстом
4. Функция `loadTasks(filename string) ([]Task, error)`:
   - Читает через `os.ReadFile`
   - Если файла **нет** — возвращает **пустой слайс** и **nil ошибку**
   - Если другая ошибка чтения — возвращает обёрнутую ошибку
   - Парсит JSON через `Unmarshal`
   - Если ошибка парсинга — возвращает обёрнутую ошибку
5. В `main`:
   - Создаёт слайс из 3 задач, сохраняет в `test.json`
   - Загружает обратно, выводит
   - Пытается загрузить `nonexistent.json`, выводит результат

## Шаги

1. Создай папку `practice-files/`, инициализируй модуль
2. Скопируй тип `Task` с тегами из модуля 1
3. Напиши `saveTasks` — `Marshal` + `WriteFile` с обёртыванием ошибок
4. Напиши `loadTasks` со специальной обработкой `os.ErrNotExist` через `errors.Is`
5. Напиши `main`, который вызывает обе функции и выводит результат
6. Запусти, проверь, что файл создан (`cat test.json` или открой в редакторе)
7. Проверь, что несуществующий файл обрабатывается корректно

## Эксперименты

1. **Удали `test.json` руками и запусти программу снова.** Она должна работать без ошибок — `saveTasks` создаст файл заново.

2. **Проверь, что WriteFile перезаписывает целиком:**
   - Создай слайс из 3 задач, сохрани
   - Создай слайс из 1 задачи, сохрани **в тот же файл**
   - Открой файл — там должна быть только 1 задача, никаких «остатков» от 3
   - Это и есть «полная перезапись»

3. **Поменяй права с `0644` на `0600`** и запусти. Файл всё ещё работает? На Linux/macOS проверь права командой `ls -l test.json` — увидишь `rw-------` вместо `rw-r--r--`.

4. **Сломай JSON в файле руками.** Открой `test.json`, удали закрывающую скобку, сохрани. Запусти программу — что произойдёт? Какое сообщение об ошибке?

5. **Используй `Errorf` без `%w`:** замени `fmt.Errorf("ошибка чтения: %w", err)` на `fmt.Errorf("ошибка чтения: %s", err)`. Сравни. На первый взгляд то же самое — но теперь у тебя **нет связи** с оригинальной ошибкой, и `errors.Is` не сможет её найти. Это важная тонкость для следующих проектов.

6. **Проверь errors.Is для несуществующего файла:**
   ```go
   _, err := os.ReadFile("really-no-file.json")
   fmt.Println("err:", err)
   fmt.Println("is not exist:", errors.Is(err, os.ErrNotExist))
   ```
   Должно вывести `true`.

7. **Проверь errors.Is для битого JSON:**
   ```go
   _, err := loadTasks("broken.json")  // файл с битым JSON
   fmt.Println("is not exist:", errors.Is(err, os.ErrNotExist))
   ```
   Должно вывести `false` — это не «нет файла», это другая ошибка.

Это важный момент: одна и та же техническая ситуация может быть **ошибкой** в одном контексте и **нормальным состоянием** в другом. Дизайн API — это в том числе решение, что считать ошибкой.

И ещё спроси: «что случится с моим кодом, если завтра я захочу, чтобы "файла нет" наоборот считалось ошибкой и программа спрашивала бы пользователя "создать новый список?"?». Сколько мест придётся поправить? Если только `loadTasks` — отлично, ты сделал правильное **разделение**: политика принятия решения локализована в одной функции.
