# Модуль 2. Слайс задач — теория

## Контекст

В модуле 1 ты научился создавать **одну** задачу. Но настоящий to-do лист — это **много** задач, и их количество не известно заранее. Тебе нужен способ хранить и обрабатывать **переменное количество** однотипных значений.

В Go для этого есть **слайсы** (slices). Слайс — это, наверное, самая важная и самая концептуально сложная структура данных в первых проектах. Если ты разберёшься с ней по-настоящему, остальные коллекции в Go покажутся очевидными.

## Ядро: что такое слайс

Слайс — это **динамический список значений одного типа**. «Динамический» значит, что его длина может расти и уменьшаться по ходу работы программы.

Объявление и создание:

```go
var tasks []Task              // пустой слайс задач (длина 0)
tasks := []Task{}             // то же самое, через короткий синтаксис
tasks := make([]Task, 0)      // явное создание через make
tasks := []Task{              // создание сразу с тремя элементами
    {ID: 1, Title: "Купить молоко"},
    {ID: 2, Title: "Позвонить маме"},
    {ID: 3, Title: "Сделать домашку"},
}
```

Все четыре формы валидны. Самые частые — первая (когда заполняешь по ходу) и четвёртая (когда знаешь данные сразу).

## Доступ к элементам и длина

К элементу обращаются по индексу — как в массивах большинства языков. Индексы начинаются с **нуля**.

```go
tasks := []Task{
    {ID: 1, Title: "первая"},
    {ID: 2, Title: "вторая"},
}

fmt.Println(tasks[0].Title)  // "первая"
fmt.Println(tasks[1].Title)  // "вторая"

fmt.Println(len(tasks))      // 2
```

`len(slice)` возвращает количество элементов. Это безопасный способ узнать, сколько данных в слайсе, перед тем как лезть по индексу.

**Главная ловушка:** если ты обратишься к индексу, которого **нет** (`tasks[5]` для слайса из 2 элементов), программа упадёт с паникой `index out of range`. Это не «варнинг», это аварийное завершение. Всегда проверяй длину, прежде чем лезть по индексу из ненадёжного источника.

## Добавление элементов: append

Чтобы добавить элемент в слайс, используют функцию `append`:

```go
tasks := []Task{}

newTask := Task{ID: 1, Title: "Купить молоко"}
tasks = append(tasks, newTask)
```

**Очень важно**: `append` **возвращает новый слайс**, который надо присвоить обратно. Если ты напишешь просто `append(tasks, newTask)` без присваивания — изменения потеряются. Это самая частая ошибка новичка.

```go
// НЕ работает — добавление потеряется
append(tasks, newTask)

// Работает — присваиваем обратно
tasks = append(tasks, newTask)
```

Можно добавить несколько элементов сразу:

```go
tasks = append(tasks, task1, task2, task3)
```

Можно добавить весь другой слайс через `...`:

```go
tasks = append(tasks, otherTasks...)
```

Три точки после имени слайса — это «распакуй слайс в отдельные аргументы». Без них Go подумает, что ты пытаешься добавить **сам слайс** как один элемент, и это не скомпилируется.

## Перебор слайса: range

Чтобы пройти по всем элементам, используют `for ... range`:

```go
for i, task := range tasks {
    fmt.Printf("%d: %s\n", i, task.Title)
}
```

`range` возвращает **два значения** на каждой итерации:

- `i` — индекс элемента (`0`, `1`, `2`, ...)
- `task` — **копия** элемента

Если индекс не нужен — заменяй на `_`:

```go
for _, task := range tasks {
    fmt.Println(task.Title)
}
```

Если нужен только индекс — можно опустить второй параметр:

```go
for i := range tasks {
    fmt.Println(i)
}
```

## ⚠️ Важная ловушка: range даёт копию

Запомни эту строчку и перечитай её три раза: **`range` копирует каждый элемент в переменную итерации**. Если ты изменишь `task` внутри цикла, **оригинал в слайсе не изменится**.

```go
// НЕ работает — изменения теряются
for _, task := range tasks {
    task.Done = true  // меняет копию, не оригинал
}
// после цикла все Done всё ещё false

// Работает — обращаемся напрямую по индексу
for i := range tasks {
    tasks[i].Done = true
}
// после цикла все Done == true
```

Это **прямое продолжение** того, что ты узнал в модуле 1 про receiver-значения. Та же история: копия vs оригинал. Раз понял там — поймёшь и здесь.

## Внутреннее устройство слайса (упрощённо)

Это не для зубрёжки, но для интуиции — крайне полезно.

Слайс внутри — это **три поля**:

1. **Указатель** на начало хранилища (некий массив в памяти)
2. **Длина** (`len`) — сколько элементов сейчас занято
3. **Ёмкость** (`cap`) — сколько элементов влезет, прежде чем потребуется выделить новую память

```
slice = { ptr: 0x1000, len: 3, cap: 8 }

память: [task1][task2][task3][   ][   ][   ][   ][   ]
         ^ ptr  ----- len -----  ----- cap -----
```

Когда ты делаешь `append`, Go проверяет: «есть ли место в текущем хранилище (cap > len)?». Если есть — просто записывает новый элемент и увеличивает len. Если нет — **выделяет новый, более крупный массив**, копирует туда старые данные, добавляет новый элемент, и слайс начинает указывать на новое место.

Зачем тебе это знать? Чтобы понимать: **`append` иногда возвращает слайс, указывающий на другое место в памяти**. Поэтому его и надо присваивать обратно — старая переменная-слайс по-прежнему смотрит на старое место.

Поэтому же бывают «странные» баги, когда два слайса делят одну память — но об этом узнаешь позже, когда потребуется.

## Полный пример

```go
package main

import "fmt"

type Task struct {
    ID    int
    Title string
    Done  bool
}

func (t Task) String() string {
    status := "[ ]"
    if t.Done {
        status = "[x]"
    }
    return fmt.Sprintf("%d. %s %s", t.ID, status, t.Title)
}

func main() {
    // Создаём пустой слайс
    var tasks []Task

    // Добавляем элементы через append
    tasks = append(tasks, Task{ID: 1, Title: "Купить молоко"})
    tasks = append(tasks, Task{ID: 2, Title: "Позвонить маме"})
    tasks = append(tasks, Task{ID: 3, Title: "Сделать домашку"})

    fmt.Println("Всего задач:", len(tasks))
    fmt.Println("---")

    // Перебираем и выводим
    for _, t := range tasks {
        fmt.Println(t.String())
    }

    fmt.Println("---")

    // Отмечаем первую как выполненную (через индекс!)
    tasks[0].Done = true

    // Выводим снова
    for _, t := range tasks {
        fmt.Println(t.String())
    }
}
```

Вывод:

```
Всего задач: 3
---
1. [ ] Купить молоко
2. [ ] Позвонить маме
3. [ ] Сделать домашку
---
1. [x] Купить молоко
2. [ ] Позвонить маме
3. [ ] Сделать домашку
```
